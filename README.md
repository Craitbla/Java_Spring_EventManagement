# EventManagement

Этот проект реализует систему для управления мероприятиями с использованием **Java Spring Boot**, **PostgreSQL**, и **Docker**. Он включает создание и управление клиентами, мероприятиями, бронированиями билетов и работой с базой данных.

## Стек технологий

* **Backend**: Java 21, Spring Boot
* **Database**: PostgreSQL
* **Тестирование**: JUnit, Mockito, Testcontainers
* **Сборка и запуск**: Maven, Docker, Docker Compose

## Описание

Проект включает несколько основных компонентов:

* **Система управления мероприятиями**, где можно управлять клиентами, мероприятиями и бронированиями.
* **Тестирование**: все тесты запускаются через Maven с использованием **Testcontainers**, которые автоматически настраивают базы данных и запускают тесты в изолированном контейнере.
* **Docker**: проект можно запустить в контейнерах с использованием Docker и Docker Compose. Вся инфраструктура, включая базу данных и приложение, работает в контейнерах, что позволяет легко запускать и разворачивать систему на любых устройствах.

## Запуск проекта

1. **Клонировать репозиторий**:

   ```bash
   git clone git@github.com:Craitbla/Java_Spring_EventManagement.git
   cd EventManagement
   ```

2. **Собрать и запустить проект с использованием Docker**:
   Для того чтобы запустить проект с помощью Docker Compose, выполните следующую команду:

   ```bash
   docker-compose up --build
   ```

   Это создаст два контейнера:

    * Приложение (с Spring Boot проектом)
    * Базу данных (PostgreSQL)

3. **Конфигурация базы данных**:
   В **`docker-compose.yml`** настроена база данных PostgreSQL с пользователем и паролем. Приложение будет использовать эти данные для подключения. Все настройки базы данных для продакшн профиля находятся в файле **`application-prod.properties`**.

   Для изменения настроек базы данных вам нужно будет обновить переменные окружения в **`docker-compose.yml`** (например, **`POSTGRES_USER`**, **`POSTGRES_PASSWORD`**, и **`POSTGRES_DB`**) и при необходимости в **`application-prod.properties`**. Эти параметры будут использоваться для подключения к базе данных.

4. **Порт приложения**: после запуска проект будет доступен на порту **8080**.

## Тестирование

1. Тесты выполняются автоматически при запуске с использованием **JUnit** и **Mockito**.
2. Для тестов используется **Testcontainers**, который создает изолированные контейнеры для базы данных PostgreSQL.
3. Результаты тестов можно найти в директориях **`target/surefire-reports`** (отчет по тестам) и **`target/jacoco`** (покрытие кода).
4. **Покрытие кода**: тесты покрывают большинство ключевых сценариев работы приложения, что позволяет убедиться в корректности его функционала и надежности. Покрытие кода можно найти в отчете **Jacoco**, доступном в папке **`target/jacoco`**.
